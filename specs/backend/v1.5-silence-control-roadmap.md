# v1.5 — Silence & control

## North star

KnotTrace remains a background companion users can **leave running**. They choose when the OS speaks, see **what automated actions did**, and reverse them without digging through OS settings.

Preserves existing principles:

- observe-first
- reversible assists
- low background overhead
- notifications are an opt-in interruption, not the product

## Theme (locked)

**Silence & control** — not platform parity, not proxy depth, not mobile polish.

| In scope | Out of scope (defer) |
|----------|----------------------|
| Ship Do Not Disturb | macOS/Linux DNS apply |
| Tray DND toggle | Watchlist domains |
| Auto-protect audit log | DIRECT-node fallback UX |
| Optional notification digest | Mobile signing / real mobile UX |
| Protect page “quiet controls” clarity | Background throughput, MTU auto-fix |

## Product outcomes

1. User can silence **all** OS notifications in one switch (app + tray).
2. Monitoring, in-app Protect alerts, and auto-protect continue under DND.
3. Every auto DNS / proxy action leaves an auditable, human-readable trail with outcome.
4. User can open Protect and understand quiet vs notify vs auto-apply without reading docs.

### Phase B — Recovery audit log (v1.5.0 core)

Append-only log of automated protect actions (not manual Assist clicks, unless we later unify).

**Store:** `%LOCALAPPDATA%/KnotTrace/auto_protect_log.jsonl` (cap ~200 lines / rotate)

**Record fields (minimum):**

| Field | Notes |
|-------|-------|
| `timestamp` | UTC |
| `kind` | `dns` \| `connect` |
| `success` | bool |
| `message` | existing action message |
| `trigger` | e.g. untrusted, integrity, site_access, poor_grade |
| `check_reason` | `scheduled` / `network_change` / … |
| `rollback_hint` | short string (“Restore DNS Assist”, “Re-open Connect”) |

**API:**

- `list_auto_protect_log(limit) -> Vec<AutoProtectLogEntry>`
- append inside `run_auto_protect` after applied/failed actions (even when DND on)

**UI:** Protect page section **Recent automatic actions** (newest first, empty state when none).

**Exit:** At least one successful auto DNS/connect path writes a row; UI lists it; restore path remains existing Assist/Connect UI (no new rollback engine in B).

## Non-goals

- Silencing in-app Protect **alerts** (dashboard stays the source of truth)
- Disabling the health monitor via DND
- Push/email/webhook channels
- Full change-management / undo stack beyond existing Assist restore + Connect re-switch
- Changing auto-protect trigger policy (separate from silence)

## Architecture notes

| Concern | Owner |
|---------|-------|
| DND flag + `should_notify` | `network-core::protect` |
| OS notify + auto-protect runtime | `apps/desktop/src-tauri/src/protect.rs` |
| Tray toggle | `apps/desktop/src-tauri/src/tray.rs` |
| Audit persistence | `network-core` (preferred) or thin Tauri helper using `data_dir()` |
| Protect UI | `ProtectPage.tsx` |

Tray toggle must call `save_protect_settings` then emit `protect-status-updated` (or a dedicated `protect-settings-updated`) so the open window stays in sync.
